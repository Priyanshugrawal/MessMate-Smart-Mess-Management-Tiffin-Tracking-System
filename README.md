# MessMate - Smart Mess Management & Tiffin Tracking System

Production-ready full-stack web app for mess owners and students with role-based dashboards, meal management, skip tracking, payment records, complaints, suggestions, ratings, and realtime chat/delivery updates.

## Tech Stack

- Frontend: React + Vite + Tailwind CSS + Axios + React Router
- Backend: Node.js + Express + Socket.io
- Database: MySQL
- Auth: JWT + bcrypt
- Maps: Google Maps embed/API key ready via env

## Features Implemented

- JWT authentication with roles: `admin`, `student`
- Secure protected API routes and role guards
- Admin dashboard metrics + analytics charts
- Menu management (add/update by date, image URL/upload support)
- Student management (add/remove/list)
- Skip meal with 3-hour cutoff enforcement and date-range support
- Transaction history + admin payment/refund endpoints
- Complaint submission and resolution tracking
- Food suggestion collection and top-requested view
- Meal rating (taste/quantity/quality) + aggregated summary
- Delivery status + realtime location updates with Socket.io
- Realtime group/private chat with stored message history
- Responsive UI for mobile and desktop

## Database Setup (MySQL)

1. Create DB schema:

```sql
SOURCE backend/database/schema.sql;
```

2. Create initial admin user (example hash generated by app is bcrypt; create via API `/api/auth/register`):

```json
{
  "name": "Mess Owner",
  "email": "admin@messmate.com",
  "phone": "9999999999",
  "password": "Admin@123",
  "role": "admin",
  "plan_type": "monthly"
}
```

## Environment Variables

Backend: copy `backend/.env.example` to `backend/.env`

Frontend: copy `frontend/.env.example` to `frontend/.env`

Set your values for DB, JWT, API URLs, and Google Maps key.

## Run Locally

### Backend

```bash
cd backend
npm install
npm run dev
```

### Frontend

```bash
cd frontend
npm install
npm run dev
```

Frontend defaults to Vite port `5173`, backend to `5000`.

## API Overview

Base URL: `http://localhost:5000/api`

- Auth: `POST /auth/register`, `POST /auth/login`, `GET /auth/me`
- Admin: `GET /admin/dashboard`, `GET /admin/analytics`
- Menu: `GET /menu`, `GET /menu/today`, `POST /menu`
- Users: `GET /users/students`, `POST /users/students`, `DELETE /users/students/:id`
- Skip meals: `GET /skip-meals`, `POST /skip-meals`
- Transactions: `GET /transactions`, `POST /transactions`, `POST /transactions/refund`
- Complaints: `GET /complaints`, `POST /complaints`, `PATCH /complaints/:id`
- Suggestions: `GET /suggestions`, `POST /suggestions`
- Ratings: `GET /ratings`, `POST /ratings`
- Delivery: `GET /delivery/latest`, `POST /delivery/update`
- Chat: `GET /chat/messages?room=general`

## Deployment

### Frontend (Vercel / Netlify)

- Build command: `npm run build`
- Output dir: `dist`
- Env: `VITE_API_URL`, `VITE_API_ORIGIN`, `VITE_SOCKET_URL`, `VITE_GOOGLE_MAPS_API_KEY`

### Backend (Render / Railway)

- Start command: `npm start`
- Env: all variables from `backend/.env.example`
- Persistent disk for uploads if needed

### MySQL

- Use managed MySQL (Railway, PlanetScale, Aiven, RDS, etc.)
- Update backend DB env vars with production credentials

## Validation and Security Notes

- Password hashing with bcrypt
- JWT auth middleware on protected routes
- Request validation on auth routes
- Centralized error handler
- CORS configurable via `CLIENT_URL`
